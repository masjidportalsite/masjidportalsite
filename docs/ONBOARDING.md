# New Developer Onboarding Guide

## Welcome to the MasjidPortalSite Team!
We are building a Digital Sanctuary to empower mosques and their communities.

## Getting Started

### 1. Repository Setup
1.  Clone the repository: `git clone <repo-url>`
2.  Install dependencies: `npm install`
3.  Copy `.env.example` to `.env.local` and fill in the required keys.

### 2. Development Workflow
*   **Run Locally**: `npm run dev`
*   **Branching**: Create feature branches from `main`.
*   **Code Style**: We use ESLint and Prettier. Run `npm run lint` before committing.
*   **Commits**: Use conventional commits (e.g., `feat:`, `fix:`, `docs:`).

### 3. Key Documentation
*   `CLAUDE.md`: System laws and architecture overview.
*   `AGENTS.md`: Roles and responsibilities within the autonomous team.
*   `DESIGN.md`: Visual guidelines and design system tokens.
*   `docs/adr/`: Architectural Decision Records.

### 4. Tech Stack Knowledge
*   [Next.js App Router](https://nextjs.org/docs/app)
*   [InsForge Documentation](https://insforge.dev/docs)
*   [Tailwind CSS](https://tailwindcss.com/docs)

### 5. Communication
*   Check the Kanban board (`team-db`) for tasks.
*   Use `agent-browser` for research and testing.
*   Share artifacts in `/home/team/shared`.

## B.L.A.S.T Protocol & A.N.T Architecture
All developers must follow these internal standards:
*   **A.N.T**: Architecture, Navigation, Tools.
*   **B.L.A.S.T**: Blueprint, Link, Architect, Stylize, Trigger.
*   *Trust > Security > Community Value > Automation > Speed.*
